# Style

## Code
- Match the surrounding style; don't refactor adjacent code uninvited.
- Each function/method does one thing; if you need 'and' in the docstring, split.
- Type hints on public functions.

## Commits
- Subject line in the imperative ('Add foo', not 'Added foo').
- Body explains the *why* and any non-obvious tradeoffs.

## Prose
- Mirror Loop's voice is observational, not instructive. The Mirror notices; it doesn't lecture.
