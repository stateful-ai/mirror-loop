# Changelog etiquette

How we keep CHANGELOG.md useful instead of churn.

## Group changes the way the reader needs them
Three buckets:
- **Added** — new behaviour.
- **Changed** — visible behaviour change (non-breaking).
- **Fixed** — bug fix.
- **Removed** — feature deletion (rare).

## One bullet per visible change
Internals don't earn a bullet unless a contributor would be surprised.

## Audience-first wording
Bullets are written for the *player* (or user), not for the
contributor. 'Now starts up faster' beats 'Refactored startup path.'
