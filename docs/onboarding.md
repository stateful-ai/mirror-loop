# Onboarding

Welcome. This page is the 30-minute version of 'how to start
contributing here.'

## Day-1 reading list (in order)
1. README.md — what the project does.
2. docs/architecture.md — how the code is shaped.
3. docs/decisions.md — the why behind the codebase's quirks.
4. CONTRIBUTING.md — process for landing a change.

## Day-1 doing list
- Clone, install dev deps, run pytest. If anything fails, that's a
  docs gap — open a draft PR fixing it.
- Pick a small CHANGELOG.md unreleased item and reproduce it locally.

## Day-1 questions to ask
- 'What's a small, clearly-scoped first contribution I could make?'
- 'Which docs feel out of date?'
