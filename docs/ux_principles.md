# UX principles

How the project speaks to whoever runs it.

## Observation, not instruction
Tell the user what you noticed; don't tell them what to do. The
Mirror metaphor — let the meaning land for them.

## Numbers do the heavy lifting
In any narrative output, prefer one concrete number over three
adjectives.

## Silence is fine
An empty result is still a useful one. Don't pad output to look busy.

## No fake humor, no fake urgency
Don't write 'Oops!' or 'Hurry!' — they age badly and erode trust.
